# 💣 مسابقة الألغام — Team Trivia "Bluff" Game (على غرار Fibbage)

لعبة أسئلة جماعية مباشرة بنمط **اللغم**: يظهر السؤال **بلا خيارات**، فتكتب كل مجموعة
**جوابًا خاطئًا مضلِّلًا** (لغمًا)، ثم تظهر كل الألغام مخلوطة مع **الجواب الصحيح** لتختار
منها المجموعات. من يصيب الصحيح يكسب، ومن يوقِع غيره في لغمه يكسب أيضًا! ثلاث واجهات
متزامنة لحظيًا: **شاشة عرض** للجمهور، **شاشة للمقدّم** للتحكم، و**جوال لكل مجموعة**.

> تطبيق واحد قائم بذاته: `npm install && npm start` — بدون قاعدة بيانات وبدون خطوة build.

---

## 🚀 التشغيل السريع

```bash
npm install
npm start
```

ثم افتح في المتصفح:

| الواجهة | الرابط | لمن؟ |
|---|---|---|
| 📺 **شاشة العرض** | `http://localhost:3000/display` | البروجكتر / الشاشة الكبيرة |
| 🎛️ **شاشة المقدّم** | `http://localhost:3000/admin` | المقدّم (رمز الدخول الافتراضي `1234`) |
| 📱 **جوال المجموعة** | `http://localhost:3000/player` | المجموعات (دخول بكود أو QR) |

### متغيّرات البيئة

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `PORT` | `3000` | منفذ الخادم |
| `ADMIN_PIN` | `1234` | رمز دخول المقدّم — **غيّره دائمًا** |
| `BASE_URL` | (اكتشاف تلقائي) | الرابط الأساسي لتوليد روابط/QR المجموعات |

```bash
ADMIN_PIN=7788 PORT=3000 npm start
```

## 🖥️ التشغيل المحلي (Wi-Fi + بروجكتر)

1. شغّل الخادم على اللابتوب المتصل بالبروجكتر.
2. افتح `/display` على البروجكتر و`/admin` على شاشة المقدّم.
3. تأكّد أن هواتف المشاركين على **نفس شبكة Wi-Fi**.
4. الروابط ورموز QR للمجموعات تُبنى تلقائيًا من عنوان الجهاز على الشبكة.
   إن لزم، اضبط الرابط الأساسي يدويًا من تبويب **«الروابط»** (مثال: `http://192.168.1.10:3000`).

## ☁️ التشغيل السحابي (استضافة)

يعمل مباشرة على منصّات مثل Render / Railway / أي خادم Node — راجع `DEPLOY.md`:

- أمر التشغيل: `npm start` — يستمع على `PORT` الذي توفّره المنصّة.
- يكتشف الرابط العام تلقائيًا من ترويسات البروكسي (`X-Forwarded-Host`)، أو اضبط `BASE_URL`.
- اضبط `ADMIN_PIN` بقيمة قوية في إعدادات المنصّة.

## 🎮 كيف تُلعب؟ (المشترك يصوّر QR ويكتب اسمه ويدخل)

1. من **شاشة المقدّم** ← «الإعداد»: أنشئ مسابقة (اسمها، مدة كل مرحلة، نقاط السؤال، عدد
   الأسئلة من المكتبة، مكافأة السرعة). الأسئلة تُختار **عشوائيًا** من مكتبة كبيرة.
2. المشتركون يصوّرون **رمز QR واحدًا** (على شاشة العرض/المقدّم)، يكتبون اسم مجموعتهم، ويدخلون.
3. اضغط **«ابدأ اللعبة»**. لكل سؤال ثلاث مراحل:
   - **① كتابة اللغم:** يظهر السؤال بلا خيارات، وتكتب كل مجموعة **جوابًا خاطئًا مضلِّلًا**.
   - **② الاختيار:** اضغط **«اعرض الخيارات»** — تظهر كل الألغام + الجواب الصحيح مخلوطة،
     وتختار كل مجموعة إجابة (ولا تستطيع اختيار لغمها).
   - **③ الكشف:** اضغط **«اكشف الإجابة»** — يظهر الصحيح، وصاحب كل لغم، ومن وقع فيه، وتُحتسب النقاط.
4. اضغط **«التالي»** للسؤال التالي. بعد الأخير تظهر منصّة التتويج للثلاثة الأوائل.

> **إضافة مجموعة أثناء اللعب:** أي مجموعة تصوّر QR وتنضم في أي لحظة — بنقاط صفر ومن السؤال الحالي.

---

## 🏆 احتساب النقاط (نمط اللغم)

- **إصابة الجواب الصحيح:** نقاط السؤال كاملة (مع **مكافأة سرعة** اختيارية: 50%–100%).
- **إيقاع مجموعة أخرى في لغمك:** تكسب صاحبة اللغم **نصف نقاط السؤال** عن كل مجموعة وقعت فيه.
- إن تطابق لغمان لمجموعتين، يُدمجان في خيار واحد ويكسب صاحباه معًا عند وقوع غيرهما فيه.
- لا يُحتسب على من وقع في لغم أي خصم. ويمكن للمقدّم منح/خصم نقاط يدويًا.

---

## 👁️ ما يراه كل طرف

- **جوال المجموعة:** اسمها ونقاطها وترتيبها؛ في «كتابة اللغم» حقلٌ لكتابة جواب مضلِّل؛ في
  «الاختيار» أزرار بكل الخيارات (لغمها معطّل)؛ وبعد الكشف نتيجتها وكم أوقع لغمها.
- **شاشة العرض:** السؤال والمؤقّت وعدد المجيبين، ثم الخيارات، ثم الكشف (الصحيح + صاحب كل
  لغم + عدد من وقع) وشريط الترتيب، ومنصّة التتويج. (لا تكشف شيئًا قبل أوانه.)
- **شاشة المقدّم:** كل التفاصيل — الجواب الصحيح دائمًا أمامه، الألغام المكتوبة وأصحابها،
  الخيارات والاختيارات، الترتيب المباشر، وإدارة كاملة للأسئلة والمجموعات.

## ✨ المميّزات

- نمط «اللغم» الممتع: كل مجموعة تصنع فخًّا للبقية، فيصير التفكير مزدوجًا.
- انضمام ذاتي: المشترك يصوّر **QR واحدًا**، يكتب اسم مجموعته، ويدخل فورًا.
- واجهة عربية (RTL) وتصميم احتفالي، ومزامنة لحظية عبر Socket.IO.
- خيارات ملوّنة، مؤقّت دائري، ومؤثرات صوتية خفيفة مُولّدة برمجيًا (بلا ملفات).
- مكتبة أسئلة عربية ضخمة (آلاف الأسئلة) تُختار منها عشوائيًا عند كل مسابقة.
- إدارة كاملة للأسئلة (نص + الجواب الصحيح) والمجموعات والنقاط من شاشة المقدّم.
- **إضافة مجموعات في أي وقت** — حتى في منتصف اللعب.
- **حفظ تلقائي** لحالة المسابقة (`data/competition.json`) للتعافي بعد إعادة التشغيل.

## 🧱 البنية التقنية

```
server/
  index.js                # إقلاع Express + Socket.IO + الملفات الثابتة
  engine/
    GameEngine.js         # آلة الحالة: المراحل الثلاث، المؤقّت، الألغام، احتساب النقاط
    scoring.js            # احتساب نقاط الإصابة + مكافأة السرعة
    Competition.js        # نموذج المسابقة (أسئلة/ألغام/خيارات/مجموعات/ترتيب)
    seedQuestions.js      # واجهة تحميل المكتبة
    questions.json        # مكتبة الأسئلة (نص + الجواب الصحيح + التصنيف)
  net/
    sockets.js            # غرف Socket.IO + بثّ مُفلتر حسب الدور + أوامر المقدّم/المجموعات
    routes.js             # REST: الإعدادات، الانضمام، توليد QR، الرابط الأساسي
    views.js              # بناء الحمولة الخاصة بكل دور
  state/persist.js        # حفظ/تحميل لقطة JSON
public/
  display/  admin/  player/  shared/   # الواجهات الثلاث + الموارد المشتركة
test/engine.test.js       # اختبارات محرّك اللعبة
```

**التقنيات:** Node.js · Express · Socket.IO · qrcode · واجهات HTML/CSS/JS خام (بلا إطار عمل).

## 📚 مكتبة الأسئلة

يأتي المشروع بمكتبة أسئلة عربية ضخمة (`server/engine/questions.json`) عبر تصنيفات متعددة
(ثقافة عامة، قرآن، سيرة، جغرافيا، ألغاز، تاريخ...). عند إنشاء مسابقة يُختار منها عدد
عشوائي (يحدّده المقدّم)، ويمكن إضافة/تحرير/حذف أي سؤال من تبويب **«الأسئلة»**.
كل سؤال = نص + الجواب الصحيح فقط (الخيارات تُصنع من ألغام المجموعات أثناء اللعب).
