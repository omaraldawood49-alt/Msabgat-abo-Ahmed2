# 📈 بورصة رواحل — Educational Stock Market Competition

لعبة جماعية تعليمية تحاكي سوق الأسهم بطريقة مبسطة وممتعة، تهدف إلى تعليم المشاركين
مفاهيم **العرض والطلب**، و**اتخاذ القرار**، و**إدارة رأس المال**، و**العمل الجماعي** —
عبر منافسة مباشرة بين المجموعات. الأسعار محاكاة ذكية (لا تعتمد على أسعار حقيقية)
تتغيّر حسب العرض والطلب أو حسب إعدادات المشرف.

> تطبيق واحد قائم بذاته: `npm install && npm start` — بدون قاعدة بيانات وبدون خطوة build.

---

## 🚀 التشغيل السريع

```bash
npm install
npm start
```

ثم افتح في المتصفح:

| الواجهة | الرابط | لمن؟ |
|---|---|---|
| 📊 **شاشة العرض** | `http://localhost:3000/display` | البروجكتر / الشاشة الكبيرة |
| 🎛️ **لوحة الأدمن** | `http://localhost:3000/admin` | المشرف (رمز الدخول الافتراضي `1234`) |
| 📱 **المتسابق** | `http://localhost:3000/player` | المجموعات (دخول بكود أو QR) |

### متغيّرات البيئة

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `PORT` | `3000` | منفذ الخادم |
| `ADMIN_PIN` | `1234` | رمز دخول الأدمن — **غيّره دائمًا** |
| `BASE_URL` | (اكتشاف تلقائي) | الرابط الأساسي لتوليد روابط/QR المجموعات |

```bash
ADMIN_PIN=7788 PORT=3000 npm start
```

## 🖥️ التشغيل المحلي (Wi-Fi + بروجكتر)

1. شغّل الخادم على اللابتوب المتصل بالبروجكتر.
2. افتح `/display` على البروجكتر و`/admin` على شاشة المشرف.
3. تأكّد أن هواتف المشاركين على **نفس شبكة Wi-Fi**.
4. الروابط و رموز QR للمجموعات تُبنى تلقائيًا من عنوان الجهاز على الشبكة.
   إن لزم، اضبط الرابط الأساسي يدويًا من تبويب **«الروابط»** (مثال: `http://192.168.1.10:3000`).

## ☁️ التشغيل السحابي (استضافة)

يعمل مباشرة على منصّات مثل Render / Railway / أي خادم Node:

- أمر التشغيل: `npm start` — يستمع على `PORT` الذي توفّره المنصّة.
- يكتشف الرابط العام تلقائيًا من ترويسات البروكسي (`X-Forwarded-Host`)، أو اضبط `BASE_URL`.
- اضبط `ADMIN_PIN` بقيمة قوية في إعدادات المنصّة.

## 🎮 كيف تُلعب؟

1. من **لوحة الأدمن**: أنشئ منافسة وحدّد الاسم وعدد الجولات ومدة الجولة وعدد المجموعات
   والمبلغ الابتدائي وعدد الأسهم ونمط تغيّر الأسعار.
2. عدّل الأسهم والمجموعات إن رغبت، ثم اعرض **QR/الكود** لكل مجموعة ليدخل المشاركون.
3. اضغط **«بدء»** — تبدأ الجولة ويتداول المشاركون (شراء/بيع) طوال مدتها.
4. عند انتهاء الوقت يُغلق التداول، ويتحرّك السوق بحركة احترافية (عدّ تنازلي 3-2-1)،
   ثم تظهر الأسعار الجديدة وتبدأ الجولة التالية.
5. بعد آخر جولة تُحسب **الثروة النهائية = النقد + قيمة الأسهم**، وتظهر منصّة تتويج للثلاثة الأوائل.

---

## 📊 نمطا تغيّر الأسعار

- **تلقائي:** الأسعار تتحرك حسب صافي العرض والطلب — كثرة الشراء ترفع السعر، وكثرة البيع تخفضه.
- **يدوي:** يحدّد الأدمن نسبة تغيّر كل سهم للجولات **1–10**، وتتكرّر دوريًا
  (الجولة 11 = الجولة 1، والجولة 12 = الجولة 2، وهكذا).

**قاعدة ذكية (منع الثبات):** لا يُسمح لأي سهم بالبقاء ثابتًا أكثر من جولتين متتاليتين؛
في الجولة الثالثة يُجبره النظام على الارتفاع أو الانخفاض تلقائيًا لضمان استمرار التفاعل.

---

## 👁️ ما يراه كل طرف

- **المتسابق:** اسم المجموعة، **النقد المتاح فقط**، الأسهم، أزرار الشراء والبيع.
  (لا يرى القيمة السوقية لمحفظته ولا إجمالي ثروته.)
- **الأدمن:** كل التفاصيل — النقد وقيمة المحفظة وإجمالي الثروة والترتيب والأرباح والخسائر لكل مجموعة.
- **شاشة العرض:** الأسعار واتجاهاتها، العدّ التنازلي، لوحة الترتيب المباشرة، شريط الأخبار، منصّة التتويج.

## ✨ المميّزات

- تصميم حديث يشبه شاشات البورصة العالمية + وضع داكن (Dark Mode) وواجهة عربية (RTL).
- انتقالات سلسة بين الجولات مع عدّ تنازلي دائري ومؤثرات بصرية وصوتية خفيفة (مُولّدة برمجيًا، بلا ملفات صوتية).
- شريط أخبار متحرّك يُولّد رسائل من حركة السوق تلقائيًا.
- كود دخول ورمز QR مستقل لكل مجموعة.
- إدارة كاملة للأسهم والمجموعات والأموال من لوحة الأدمن.
- **حفظ تلقائي** لحالة المنافسة (`data/competition.json`) للتعافي بعد إعادة تشغيل الخادم.

## 🧱 البنية التقنية

```
server/
  index.js                # إقلاع Express + Socket.IO + الملفات الثابتة
  engine/
    GameEngine.js         # آلة الحالة: الجولات، المؤقّت، التداول (مصدر الحقيقة)
    pricing.js            # نمطا التسعير + قاعدة منع الثبات
    Competition.js        # نموذج المنافسة (أسهم/مجموعات/محافظ/ترتيب)
    seedStocks.js         # الأسهم الافتراضية الـ25
  net/
    sockets.js            # غرف Socket.IO + بثّ مُفلتر حسب الدور + أوامر الأدمن/المتسابق
    routes.js             # REST: الإعدادات، الانضمام، توليد QR، الرابط الأساسي
    views.js              # بناء الحمولة الخاصة بكل دور
  state/persist.js        # حفظ/تحميل لقطة JSON
public/
  display/  admin/  player/  shared/   # الواجهات الثلاث + الموارد المشتركة
test/engine.test.js       # اختبارات محرّك اللعبة
```

**التقنيات:** Node.js · Express · Socket.IO · qrcode · واجهات HTML/CSS/JS خام (بلا إطار عمل).
