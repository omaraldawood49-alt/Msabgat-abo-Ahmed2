# 🕳️ لعبة الأفخاخ — Team Trap Trivia (على غرار Fibbage) بغرفٍ مستقلة

لعبة أسئلة جماعية مباشرة بنمط **الفخ**: **غرف مستقلة** — أنشئ لعبتك (تصير مضيفًا) أو انضمّ
للعبة غيرك برمزها. يظهر السؤال **للمضيف فقط** (يقرأه بصوته)، فتكتب كل مجموعة **إجابة خاطئة
مضلِّلة** (فخًا)، ثم تظهر كل الأفخاخ مخلوطة مع **الجواب الصحيح** لتختار منها المجموعات. من
يصيب الصحيح يكسب، ومن يوقِع غيره في فخه يكسب — وتظهر **خريطة الأفخاخ**: من نصب ومن وقع.

> تطبيق واحد قائم بذاته: `npm install && npm start` — بدون قاعدة بيانات وبدون خطوة build.

---

## 🚀 التشغيل السريع

```bash
npm install
npm start
```

ثم افتح **`http://localhost:3000/`** — كل شيء يبدأ من الصفحة الرئيسية:

- **إنشاء لعبة** → تصير مضيفًا وتحصل على رمز غرفة (لا يوجد رمز أدمن؛ رمز مضيف سرّي يُولَّد تلقائيًا لكل غرفة).
- **انضمام للعبة** → أدخل رمز الغرفة، اكتب اسم مجموعتك، وادخل.
- شاشة عرض مشتركة اختيارية: `/display?room=CODE` (يوفّر المضيف الرابط).

### متغيّرات البيئة

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `PORT` | `3000` | منفذ الخادم |
| `BASE_URL` | (اكتشاف تلقائي) | الرابط الأساسي لتوليد روابط/QR الانضمام |

---

## 🖥️ التشغيل المحلي (Wi-Fi + بروجكتر)

1. شغّل الخادم على اللابتوب المتصل بالبروجكتر.
2. افتح `/display` على البروجكتر و`/admin` على شاشة المقدّم.
3. تأكّد أن هواتف المشاركين على **نفس شبكة Wi-Fi**.
4. الروابط ورموز QR للمجموعات تُبنى تلقائيًا من عنوان الجهاز على الشبكة.
   إن لزم، اضبط الرابط الأساسي يدويًا من تبويب **«الروابط»** (مثال: `http://192.168.1.10:3000`).

## ☁️ التشغيل السحابي (استضافة)

يعمل مباشرة على منصّات مثل Render / Railway / أي خادم Node — راجع `DEPLOY.md`:

- أمر التشغيل: `npm start` — يستمع على `PORT` الذي توفّره المنصّة.
- يكتشف الرابط العام تلقائيًا من ترويسات البروكسي (`X-Forwarded-Host`)، أو اضبط `BASE_URL`.
- اضبط `ADMIN_PIN` بقيمة قوية في إعدادات المنصّة.

## 🎮 كيف تُلعب؟ (غرف مستقلة)

1. من **الصفحة الرئيسية**: اختر **«إنشاء لعبة»**، سمِّها، **اختر التصنيفات** (ثقافة، قرآن،
   سيرة، جغرافيا...)، وعدد الأسئلة والمدة والنقاط → «إنشاء». تصبح **مضيفًا** وتحصل على **رمز غرفة**.
2. المشاركون يفتحون **«انضمام للعبة»** ويدخلون رمز الغرفة (أو يصوّرون QR)، يكتبون اسم مجموعتهم، ويدخلون.
3. المضيف يضغط **«ابدأ اللعبة»**. لكل سؤال ثلاث مراحل يتحكّم بها المضيف يدويًا:
   - **① نصب الفخ:** السؤال يظهر **للمضيف فقط** (يقرأه بصوته)، وتكتب كل مجموعة **إجابة خاطئة مضلِّلة**.
   - **② الاختيار** (زر «اعرض الخيارات»): تظهر كل الأفخاخ + الجواب الصحيح مخلوطة، وتختار كل مجموعة (ولا تختار فخها).
   - **③ الكشف** (زر «اكشف الإجابة»): يظهر الصحيح و**خريطة الأفخاخ** (من نصب ومن وقع)، وتُحتسب النقاط.
4. زر **«التالي»** للسؤال القادم. بعد الأخير تظهر منصّة التتويج. وزر **«لعبة جديدة»** يبدأ جولة بأسئلة جديدة مع بقاء المشاركين.

> **الغرف مستقلة تمامًا:** كل لعبة معزولة برمزها ومضيفها السرّي. أي مجموعة تنضم في أي وقت — بنقاط صفر.

---

## 🏆 احتساب النقاط (نمط الفخ)

- **إصابة الجواب الصحيح:** نقاط السؤال كاملة (مع **مكافأة سرعة** اختيارية: 50%–100%).
- **إيقاع مجموعة أخرى في فخك:** تكسب صاحبة الفخ **نصف نقاط السؤال** عن كل مجموعة وقعت فيه.
- إن تطابق فخّان، يُدمجان في خيار واحد ويكسب صاحباه معًا. ولا يُخصم على من وقع في فخ.

---

## 👁️ ما يراه كل طرف

- **جوال المشارك:** لا يرى **نص السؤال** (يسمعه من المضيف) — يرى التصنيف فقط؛ يكتب فخه؛ يختار من
  الخيارات (فخه معطّل)؛ وبعد الكشف نتيجته، **ومن أوقع في فخه**، و**في فخ من وقع**.
- **شاشة العرض المشتركة** (اختيارية، برابط الغرفة): رمز الغرفة وQR والمنضمّون، ثم الخيارات، ثم
  الكشف (الصحيح + أصحاب الأفخاخ + الأعداد) والترتيب والتتويج. **بلا نص السؤال**.
- **شاشة المضيف:** كل شيء — **نص السؤال والجواب الصحيح أمامه**، الأفخاخ المكتوبة وأصحابها،
  الاختيارات، **خريطة الأفخاخ الكاملة**، الترتيب المباشر، وإدارة الأسئلة والمشاركين.

---

## ✨ المميّزات

- نمط «اللغم» الممتع: كل مجموعة تصنع فخًّا للبقية، فيصير التفكير مزدوجًا.
- انضمام ذاتي: المشترك يصوّر **QR واحدًا**، يكتب اسم مجموعته، ويدخل فورًا.
- واجهة عربية (RTL) وتصميم احتفالي، ومزامنة لحظية عبر Socket.IO.
- خيارات ملوّنة، مؤقّت دائري، ومؤثرات صوتية خفيفة مُولّدة برمجيًا (بلا ملفات).
- مكتبة أسئلة عربية ضخمة (آلاف الأسئلة) تُختار منها عشوائيًا عند كل مسابقة.
- إدارة كاملة للأسئلة (نص + الجواب الصحيح) والمجموعات والنقاط من شاشة المقدّم.
- **إضافة مجموعات في أي وقت** — حتى في منتصف اللعب.
- **حفظ تلقائي** لحالة المسابقة (`data/competition.json`) للتعافي بعد إعادة التشغيل.

## 🧱 البنية التقنية

```
server/
  index.js                # إقلاع Express + Socket.IO + مدير الغرف
  engine/
    RoomManager.js        # غرف مستقلة: كل غرفة محرّك + رمز غرفة + رمز مضيف سرّي
    GameEngine.js         # آلة الحالة: المراحل الثلاث، المؤقّت، الأفخاخ، احتساب النقاط
    scoring.js            # احتساب نقاط الإصابة + مكافأة السرعة
    Competition.js        # نموذج اللعبة + تصفية التصنيفات + بناء الخيارات/الأفخاخ
    seedQuestions.js      # واجهة تحميل المكتبة
    questions.json        # مكتبة الأسئلة (نص + الجواب الصحيح + التصنيف)
  net/
    sockets.js            # قنوات Socket.IO لكل غرفة + بثّ مُفلتر حسب الدور
    routes.js             # REST: إنشاء غرفة، التصنيفات، الانضمام، QR
    views.js              # حمولة كل دور (السؤال للمضيف فقط + خريطة الأفخاخ)
  state/persist.js        # حفظ/تحميل لقطة الغرف (JSON)
public/
  index.html              # الصفحة الرئيسية: إنشاء/انضمام
  admin/  player/  display/  shared/   # المضيف + المشارك + الشاشة المشتركة + الموارد
test/engine.test.js       # اختبارات المحرّك والغرف والتصنيفات
```

**التقنيات:** Node.js · Express · Socket.IO · qrcode · واجهات HTML/CSS/JS خام (بلا إطار عمل).

## 📚 مكتبة الأسئلة

يأتي المشروع بمكتبة أسئلة عربية ضخمة (`server/engine/questions.json`) عبر تصنيفات متعددة
(ثقافة عامة، قرآن، سيرة، جغرافيا، ألغاز، تاريخ...). عند إنشاء مسابقة يُختار منها عدد
عشوائي (يحدّده المقدّم)، ويمكن إضافة/تحرير/حذف أي سؤال من تبويب **«الأسئلة»**.
كل سؤال = نص + الجواب الصحيح فقط (الخيارات تُصنع من ألغام المجموعات أثناء اللعب).
