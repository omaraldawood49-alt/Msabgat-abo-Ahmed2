# ❓ مسابقة الأسئلة — Team Quiz Game (على غرار «كلك»)

لعبة أسئلة جماعية مباشرة على غرار **كلك / كاهوت**: أسئلة اختيار من متعدد، كل مجموعة تجيب من
جوالها، والنقاط تُحتسب فورًا (مع مكافأة للأسرع). ثلاث واجهات متزامنة لحظيًا:
**شاشة عرض** للجمهور، **شاشة للمقدّم** للتحكم، و**جوال لكل مجموعة**.

> تطبيق واحد قائم بذاته: `npm install && npm start` — بدون قاعدة بيانات وبدون خطوة build.

---

## 🚀 التشغيل السريع

```bash
npm install
npm start
```

ثم افتح في المتصفح:

| الواجهة | الرابط | لمن؟ |
|---|---|---|
| 📺 **شاشة العرض** | `http://localhost:3000/display` | البروجكتر / الشاشة الكبيرة |
| 🎛️ **شاشة المقدّم** | `http://localhost:3000/admin` | المقدّم (رمز الدخول الافتراضي `1234`) |
| 📱 **جوال المجموعة** | `http://localhost:3000/player` | المجموعات (دخول بكود أو QR) |

### متغيّرات البيئة

| المتغيّر | الافتراضي | الوصف |
|---|---|---|
| `PORT` | `3000` | منفذ الخادم |
| `ADMIN_PIN` | `1234` | رمز دخول المقدّم — **غيّره دائمًا** |
| `BASE_URL` | (اكتشاف تلقائي) | الرابط الأساسي لتوليد روابط/QR المجموعات |

```bash
ADMIN_PIN=7788 PORT=3000 npm start
```

## 🖥️ التشغيل المحلي (Wi-Fi + بروجكتر)

1. شغّل الخادم على اللابتوب المتصل بالبروجكتر.
2. افتح `/display` على البروجكتر و`/admin` على شاشة المقدّم.
3. تأكّد أن هواتف المشاركين على **نفس شبكة Wi-Fi**.
4. الروابط ورموز QR للمجموعات تُبنى تلقائيًا من عنوان الجهاز على الشبكة.
   إن لزم، اضبط الرابط الأساسي يدويًا من تبويب **«الروابط»** (مثال: `http://192.168.1.10:3000`).

## ☁️ التشغيل السحابي (استضافة)

يعمل مباشرة على منصّات مثل Render / Railway / أي خادم Node — راجع `DEPLOY.md`:

- أمر التشغيل: `npm start` — يستمع على `PORT` الذي توفّره المنصّة.
- يكتشف الرابط العام تلقائيًا من ترويسات البروكسي (`X-Forwarded-Host`)، أو اضبط `BASE_URL`.
- اضبط `ADMIN_PIN` بقيمة قوية في إعدادات المنصّة.

## 🎮 كيف تُلعب؟

1. من **شاشة المقدّم**: أنشئ مسابقة (اسمها، مدة السؤال الافتراضية، نقاط السؤال،
   عدد المجموعات، تفعيل مكافأة السرعة، وتحميل بنك أسئلة جاهز أو البدء فارغًا).
2. من تبويب **«الأسئلة»** أضِف/عدّل أسئلتك (نص + حتى 4 خيارات + تحديد الإجابة الصحيحة + الوقت + النقاط).
3. من تبويب **«المجموعات»** اعرض **QR/الكود** لكل مجموعة ليدخل الفريق من جواله.
4. اضغط **«بدء»** — يظهر السؤال على شاشة العرض وجوالات المجموعات، ويبدأ العدّ التنازلي.
5. كل مجموعة تختار إجابة **واحدة** (تُقفل بعد الاختيار). اضغط **«كشف الإجابة»** أو انتظر انتهاء الوقت:
   تظهر الإجابة الصحيحة وتوزيع الإجابات وتُحتسب النقاط ويتحدّث الترتيب.
6. اضغط **«السؤال التالي»** للمتابعة. بعد آخر سؤال تظهر منصّة التتويج للثلاثة الأوائل.

> **إضافة مجموعة أثناء اللعب:** من تبويب «المجموعات» اضغط «إضافة مجموعة» في أي لحظة —
> تنضم فورًا بنقاط صفر وتشارك من السؤال الحالي.

---

## 🏆 احتساب النقاط

- إجابة خاطئة أو عدم الإجابة = **0 نقطة**.
- إجابة صحيحة = **نقاط السؤال**، ومع تفعيل **مكافأة السرعة** يحصل الأسرع على نقاط أكثر
  ضمن نطاق **50%–100%** من نقاط السؤال (كلما بقي وقت أكثر لحظة الإجابة، زادت النقاط).
- يمكن للمقدّم **منح/خصم** نقاط يدويًا لأي مجموعة من تبويب «المجموعات».

---

## 👁️ ما يراه كل طرف

- **جوال المجموعة:** اسم المجموعة ونقاطها وترتيبها، نص السؤال، الخيارات كأزرار ملوّنة،
  ثم نتيجتها (صحيحة/خاطئة + النقاط المكتسبة) بعد الكشف. (لا يرى الإجابة الصحيحة قبل الكشف.)
- **شاشة العرض:** السؤال والخيارات والمؤقّت وعدد المجيبين، ثم الإجابة الصحيحة وتوزيع
  الإجابات وشريط الترتيب، ومنصّة التتويج في النهاية. (لا تكشف الصحيح قبل انتهاء السؤال.)
- **شاشة المقدّم:** كل التفاصيل — السؤال الحالي مع الإجابة الصحيحة، من أجاب، توزيع الإجابات،
  الترتيب المباشر بالنقاط، وإدارة كاملة للأسئلة والمجموعات والتحكم.

---

## ✨ المميّزات

- واجهة عربية (RTL) وتصميم احتفالي، ومزامنة لحظية عبر Socket.IO.
- أزرار خيارات ملوّنة بأشكال مميّزة (نمط كلك/كاهوت) على الجوال وشاشة العرض.
- مؤقّت دائري تنازلي ومؤثرات صوتية خفيفة مُولّدة برمجيًا (بلا ملفات صوتية).
- كود دخول ورمز QR مستقل لكل مجموعة.
- إدارة كاملة لبنك الأسئلة (إضافة/تحرير/حذف/ترتيب) والمجموعات والنقاط من شاشة المقدّم.
- **إضافة مجموعات في أي وقت** — حتى في منتصف اللعب.
- **حفظ تلقائي** لحالة المسابقة (`data/competition.json`) للتعافي بعد إعادة تشغيل الخادم.

## 🧱 البنية التقنية

```
server/
  index.js                # إقلاع Express + Socket.IO + الملفات الثابتة
  engine/
    GameEngine.js         # آلة الحالة: الأسئلة، المؤقّت، الإجابات، احتساب النقاط (مصدر الحقيقة)
    scoring.js            # احتساب نقاط الإجابة + مكافأة السرعة
    Competition.js        # نموذج المسابقة (أسئلة/مجموعات/ترتيب)
    seedQuestions.js      # بنك أسئلة افتراضي جاهز
  net/
    sockets.js            # غرف Socket.IO + بثّ مُفلتر حسب الدور + أوامر المقدّم/المجموعات
    routes.js             # REST: الإعدادات، الانضمام، توليد QR، الرابط الأساسي
    views.js              # بناء الحمولة الخاصة بكل دور
  state/persist.js        # حفظ/تحميل لقطة JSON
public/
  display/  admin/  player/  shared/   # الواجهات الثلاث + الموارد المشتركة
test/engine.test.js       # اختبارات محرّك اللعبة
```

**التقنيات:** Node.js · Express · Socket.IO · qrcode · واجهات HTML/CSS/JS خام (بلا إطار عمل).
